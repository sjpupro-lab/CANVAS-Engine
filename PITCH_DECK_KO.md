# **CANVAS**

### 공간 패턴 기반 AI 엔진

언어를 벡터가 아닌, 영상처럼 다룹니다.
메모리는 가중치 행렬이 아니라 한 장의 그림입니다.

---

## 문제

오늘날의 AI는 단 하나의 구조적 베팅 위에 서 있습니다:
**언어를 고정 크기의 가중치 행렬로 압축한다.**

이 베팅은 세 가지 비용을 누적시킵니다:

- **고정 파라미터** → 확장 = 재학습
- **제한된 컨텍스트** → 윈도우가 곧 메모리
- **불투명한 가중치** → 감사·수정·설명 불가

업계는 이 한계를 메우기 위해 수십억 달러를 쓰고 있습니다.
**병목은 기반 구조 그 자체입니다.**

---

## 인사이트

언어는 공간적 구조를 갖습니다.
보통 이 구조는 벡터로 투영된 뒤 가중치 안에 눌려 들어갑니다.

### CANVAS는 이 구조를 **눈에 보이게** 유지합니다.

- 텍스트 → 2D 격자 위의 밝기 패턴
- 언어 단위 → 프레임
- 프레임은 영상처럼 누적
- 메모리는 **프레임을 덧붙여** 자라남 (행렬 재학습 아님)

---

## 동작 방식

![bg right:45% 90%](visualization_1.png)

언어는 2D 격자 위의 패턴으로 인코딩됩니다.

- 비슷한 입력 → 비슷한 패턴
- 다른 입력 → 다른 패턴
- 엔진은 패턴 자체를 **직접 비교**

임베딩 없음. 어텐션 없음. 재학습 없음.

---

## 매칭 파이프라인

![bg right:45% 90%](visualization_2.png)

검색은 2단계:

1. **빠른 조대 필터** — 패턴 위에서
2. **정밀 매칭** — 역시 패턴 위에서

두 단계 모두 시각적 표현 그 자체 위에서 동작합니다.
가중치도 벡터도 아닙니다.

---

## 열리는 것

| | |
|---|---|
| **무한 파라미터** | 입력마다 새 프레임. 상한 없음. |
| **무한 컨텍스트** | RAM이 아닌 디스크 기반. |
| **해석 가능성** | 메모리 = 시각적 패턴. 눈으로 확인. |
| **증분 학습** | 새 지식 = 프레임 1장. 재학습 없음. |
| **경량성** | 스마트폰·웨어러블·임베디드 구동. |

---

## 기존 LLM과의 비교

```
                 기존 LLM                 CANVAS
  ─────────────────────────────────────────────────────
  표현          토큰 → 벡터 → 행렬        격자 패턴
  파라미터      고정 크기 행렬            무한 스택
  컨텍스트      제한된 윈도우             디스크 기반
  해석          불투명한 가중치           시각적 패턴
  학습          재학습 / 파인튜닝         프레임 1장
  풋프린트      데이터 센터               임베디드
```

> CANVAS는 대체재가 아니라 **상보재**입니다.
> LLM에게 없는 **메모리 레이어**입니다.

---

## 왜 지금인가

1. **컨텍스트 윈도우는 벽에 부딪혔습니다** — 선형 어텐션으로는 "영원히 기억"을 풀 수 없습니다.
2. **온디바이스 AI는 전략적 우선순위입니다** — OS·통신사·OEM 모두 경량 메모리 프리미티브가 필요합니다.
3. **해석 가능성은 연구에서 규제로 이동 중입니다** — 시각적 메모리는 땜질이 아니라 **구조적 답**입니다.

CANVAS는 이 세 가지를 처음부터 전제로 설계되었습니다.

---

## 시장

크고 빠르게 성장하는 세 시장 — 하나의 기반.

- **AI 메모리 / 검색 인프라** — 벡터 DB, RAG, 에이전트 메모리
- **온디바이스 AI** — 엣지, 모바일, 임베디드
- **해석 / 감사 가능 AI** — 규제 산업, 거버넌스

**웨지:** 생성보다 **패턴의 지속성**이 중요한
검색·메모리·장기 컨텍스트 워크로드.

---

## 활용 분야

- **에이전트 메모리** — 드롭인, 잊지 않음, 벡터 DB 불필요
- **온디바이스 AI** — 클라우드 재학습 없는 증분 업데이트
- **도메인 지식 저장소** — 법률·의료·엔터프라이즈, 한 프레임씩
- **개인 AI** — 사용자와 함께 자라고, 디바이스에 남음
- **해석 가능 AI 연구** — 실제로 들여다볼 수 있는 메모리 기반

---

## 모트

- **아키텍처 독창성** — 트랜스포머·SSM의 변종이 아닙니다
- **핵심 IP 비공개 보유** — 공개 저장소 = 개요, 코어 = 비공개
- **누적 데이터 우위** — 증분 메모리가 쌓일수록 패턴 라이브러리 강화
- **풋프린트 우위** — 엣지 동작은 최적화 목표가 아닌 **구조적 속성**

---

## 현황

- **코어 엔진:** 동작하는 구현, 표준 코퍼스 벤치마크 완료
- **검증:** 검색 · 재현 · 증분 메모리 태스크
- **내부 활용:** 장기 컨텍스트 메모리 · 패턴 매칭

코어 엔진은 **비공개 저장소**에서 관리됩니다.
공개 저장소 = 개요 · 컨셉 · 데모.
