# CANVAS — 라이선스 전략

> 본 문서는 CANVAS 엔진과 관련 자료에 대한 **라이선스 전략 문서**입니다.
> 법적 계약이 아니라 전략 문서이며, 실제 조건은 개별 라이선스 파일과 상업 계약서를 따릅니다.

---

## 1. 목표

라이선스 전략은 네 가지 목표를 동시에 만족시키도록 설계됩니다.

1. **코어 보호.** 엔진은 회사의 크라운 쥬얼입니다. 상업 사용을 무력화하는 형태로 자유롭게 재배포되어서는 안 됩니다.
2. **생태계 확산.** 개발자·연구자·학생이 마찰 없이 CANVAS 위에서 학습·프로토타이핑·논문화를 할 수 있어야 합니다.
3. **프로덕션 사용으로부터 가치 회수.** 특히 대규모·온디바이스·유료 서비스 뒤에서 돌아가는 상업 배포는 반드시 매출로 돌아와야 합니다.
4. **투자자 스토리의 깔끔함.** 명확한 IP 소유, 명확한 매출 경로, 의도치 않은 카피레프트 오염 제거.

## 2. 3단 라이선스 구조

CANVAS는 **3단 오픈 코어(open-core) 구조**를 사용합니다.

```
  ┌──────────────────────────────────────────────────────────────────┐
  │  Tier 3 — 상업(Commercial)                                        │
  │  ─────────────────────                                            │
  │  프로덕션 SDK · 온디바이스 · 임베디드 · 매니지드 서비스               │
  │  라이선스: CANVAS Commercial License (유상)                        │
  │  IP:      독점, 비공개 보유                                         │
  ├──────────────────────────────────────────────────────────────────┤
  │  Tier 2 — 소스 공개 (커뮤니티)                                      │
  │  ─────────────────────────────                                    │
  │  연구·평가·비상업 용도의 레퍼런스 엔진                               │
  │  라이선스: CANVAS Community License (BSL / PolyForm-NC)             │
  │  IP:      독점, 소스는 열람 가능                                     │
  ├──────────────────────────────────────────────────────────────────┤
  │  Tier 1 — 공개(Public)                                            │
  │  ──────────────                                                   │
  │  개요, 컨셉, 데모, 시각화, 어댑터                                   │
  │  라이선스: MIT                                                     │
  │  IP:      공개                                                    │
  └──────────────────────────────────────────────────────────────────┘
```

### Tier 1 — 공개 (MIT)

- **대상:** 본 저장소 — README, 피치덱, 투자자 자료, 개념 시각화, 공개되는 얇은 레퍼런스 어댑터/데모.
- **MIT 선택 이유:** 배포 마찰 0, 평가자에게 최소 법적 오버헤드. 커뮤니티에 카피레프트 리스크 없음.
- **미포함:** 코어 엔진 구현, 패턴 매칭 내부, 최적화된 프로덕션 코드.

### Tier 2 — 소스 공개 (커뮤니티)

- **채택 예정 라이선스 후보:** **Business Source License (BSL 1.1)** + 비상업 제한, 또는 **PolyForm Noncommercial 1.0.0**.
- **대상:** 연구자·개발자·디자인 파트너가 **읽고·실행하고·벤치마크하고·그 위에 빌드**할 수 있는 수준의 레퍼런스 엔진 구현. 단, 상업 배포는 불가.
- **핵심 조항:**
  - 허용: 연구, 개인 사용, 내부 평가, 학술 논문화, 비상업 오픈소스 프로젝트
  - 불가: 프로덕션 사용, SaaS 호스팅, 온디바이스 배포, 경쟁 제품 형태의 재배포
  - **자동 라이선스 전환:** 각 BSL 릴리스는 **N년** 후(기본: 4년) Apache 2.0으로 전환. 장기적 커뮤니티 신뢰를 확보하면서 단기 상업 가치를 훼손하지 않음.
- **BSL 선택 이유 (vs AGPL/GPL):**
  - AGPL은 엔터프라이즈 고객을 기피하게 만들고 임베디드/SaaS 같은 핵심 상업 패턴을 차단
  - BSL은 명시적: 비상업은 환영, 상업은 유상 라이선스 필요
  - 선례: HashiCorp, MongoDB, CockroachDB, Sentry, Redis Labs

### Tier 3 — 상업 (독점)

- **포함:**
  - 프로덕션 하드닝된 엔진 빌드
  - 최적화된 온디바이스 / 임베디드 SDK (iOS, Android, 임베디드 리눅스, RTOS)
  - 매니지드 메모리 서비스 / 호스티드 CANVAS
  - 도메인별 메모리 팩
  - 엔터프라이즈 서포트, SLA, 손해배상(indemnification)
- **라이선스:** **CANVAS Commercial License** — 배포 단위 또는 시트 단위 협의.
- **패키징:** §5 참조.

## 3. IP 소유권 및 저장소 정책

- **공개 저장소 (`canvas-engine`)** — MIT, 자유 포크 가능.
- **코어 엔진 저장소** — **비공개**, 독점. 공개 리모트로 절대 푸시하지 않음. 접근은 핵심 팀 및 계약에 묶인 파트너로 한정.
- **외부 기여(비직원)** 는 반드시 **CLA (Contributor License Agreement)** 를 체결하여 회사가 상업적으로 재라이선스할 권리를 확보. CLA 미체결 기여는 Tier 2/3에 병합 불가.
- **카피레프트 의존성** (GPL, AGPL, LGPL 정적 링크 등) 은 코어 엔진에서 금지. 빌드 타임 검사로 강제.

## 4. 상표(Trademark)

- **"CANVAS"** 및 로고는 회사의 상표입니다.
- 공개 저장소의 MIT 라이선스는 저작권만 부여하며, **상표권은 부여하지 않습니다**. 다운스트림 사용자는 MIT로 포크할 수 있지만, **"CANVAS"** 또는 혼동 가능한 이름으로 제품을 출시할 수 없습니다.
- 이는 표준 OSS 상표 분리(carve-out) 관행이며 MIT와 완전히 양립합니다(MIT는 저작권을 다루지, 상표를 다루지 않습니다).

## 5. 상업 패키징

상업 티어는 **고객 가치와 회사의 한계 비용에 정렬되는 축**으로 가격이 매겨집니다.

| 축 | 일반적 가격 신호 |
|---|---|
| **배포 표면** | 클라우드 · 온프레 · 온디바이스 · 임베디드 |
| **규모** | 저장 프레임 수 · QPS · 디바이스 수 |
| **통합 깊이** | SDK만 · 매니지드 · 화이트라벨 |
| **서포트** | 커뮤니티 · 스탠다드 · 엔터프라이즈 (24/7, SLA) |

예시 패키지 형상 (최종화 예정):

- **Startup** — 일정 매출/투자 유치액 미만 초기 기업 대상 연단위 정액
- **Production** — 사용량 기반 (프레임 / QPS / 디바이스) + 볼륨 티어
- **Enterprise** — 협의형, 온프레 · 서포트 · SLA · 손해배상 포함
- **OEM / 임베디드** — CANVAS 내장 하드웨어 출하 단위당 로열티
- **Academic / 연구** — Tier 2 하에서 무료, 논문화 조건 부여

## 6. 집행 & 컴플라이언스

- **라이선스 검증** 을 상업 SDK에 내장 (서명된 라이선스 키, 온라인 배포는 주기적 검증, 에어갭 환경은 오프라인 활성화).
- **텔레메트리는 옵트인**, 사용자 콘텐츠는 절대 수집하지 않음. 라이선스 검증 채널은 라이선스 메타데이터만 전송.
- **감사권** 은 엔터프라이즈 계약에 포함, 합리적 위반 의심 시에만 행사.
- **오픈소스 컴플라이언스(자사 측):** 코어 엔진의 모든 의존성은 병합 전 라이선스 호환성 검토.

## 7. 투자자 스토리와의 연결

- **깨끗한 IP.** 단일 회사가 코어를 소유. CLA로 외부 기여자의 권리 주장 차단. 카피레프트 오염 없음.
- **방어 가능한 모트.** BSL은 클라우드 하이퍼스케일러가 우리 소스 위에서 경쟁 매니지드 서비스를 제공하는 것을 차단.
- **듀얼 매출 모션.** Tier 1–2의 상향식 채택이 Tier 3의 하향식 상업 계약으로 연결.
- **옵셔널 오픈소싱.** BSL 일몰 조항은 단기 매출을 포기하지 않고도 "언젠가 오픈소스"라는 스토리를 확보 — 규제, 공공, 커뮤니티 내러티브에 유용.
- **엑싯 호환성.** 구조가 인수자에게 익숙 (HashiCorp, MongoDB, Confluent 패턴) — 인수 시점에 구조 해체가 필요하지 않음.

- **Tier 3 (상업):** 첫 유료 디자인 파트너 이전 법률 자문과 함께 계약 템플릿 확정 예정.

## 9. 법무 검토가 필요한 오픈 이슈

아래 항목은 실행 전 회사의 주요 관할에서 IP·상업 법률 자문의 검토가 필요합니다.

- Tier 2 최종 선택 — **BSL 1.1** vs **PolyForm Noncommercial**
- BSL **전환 시점** (기본: +4년) 및 **전환 라이선스** (기본: Apache 2.0)
- **CLA** 양식 — 개인용 · 법인용
- **상표 등록 전략** (관할, 분류)
- 온디바이스 / 임베디드 빌드에 대한 **수출 통제** 분류
- 공간 패턴 핵심 방법에 대한 **특허 전략** (방어 출원, 선행 기술 전략)
