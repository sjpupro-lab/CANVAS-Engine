# SPATIAL-PATTERN-AI (CANVAS)

> 텍스트를 256x256 픽셀 격자 위의 밝기 패턴으로 변환하여 학습하는 공간 패턴 기반 AI 엔진.
> 언어를 토큰 벡터가 아닌, 영상 프레임처럼 취급합니다.

```
 "귀여운 고양이가 밥을 먹는다."     256x256 격자 (1절 = 1프레임)
         │                        ┌────────────────────────┐
    UTF-8 바이트                   │  ·                     │
         │                        │    ·  ·                 │
   ┌─────▼──────┐                │  · · ·  ·   ·          │
   │  X = 바이트값│                │     ·    ·              │
   │  Y = 위치   │   ──────►     │  ·    ·                 │
   │  밝기 = 가중치│               │       ·  ·             │
   └─────────────┘                │  · ·       ·  ·        │
                                  └────────────────────────┘
                                   A채널: 빈도 히트맵
```

## 동작 원리

### 1. 3-레이어 비트맵 합산

하나의 절을 3장의 독립 비트맵으로 인코딩한 뒤, 투명한 종이를 덧대듯 합산합니다.

```
  레이어       대상             가중치    포착 내용
 ─────────────────────────────────────────────────────
  기본         전체 바이트         +1     모든 바이트 위치
  단어         공백 기준 단어      +2     단어 단위 강조
  형태소       명사/동사/형용사... +1     언어 구조
 ─────────────────────────────────────────────────────
  합산 = 기본 + 단어 + 형태소       (최대 밝기 = 4)
```

```
  검증: "귀여운 고양이가 밥을 먹는다."
  ────────────────────────────────────
  기본 레이어:    40 px, 최대 1, 총  40
  단어 레이어:    37 px, 최대 2, 총  74
  형태소 레이어:  37 px, 최대 1, 총  37
  합산 (1장):     40 px, 최대 4, 총 151
                                     ↑
             보존 법칙: 151 = 40 + 74 + 37  ✓
```

### 2. RGBA 채널

```
  채널  │ 역할       │ 타입    │ 설명
 ───────┼────────────┼─────────┼──────────────────────────
   A    │ 밝기       │ uint16  │ 바이트 빈도 (3레이어 합산)
   R    │ 의미       │ uint8   │ 의미적 유사도 (AI 동적 매핑)
   G    │ 기능       │ uint8   │ 품사 / 문법 기능
   B    │ 확장       │ uint8   │ 컨텍스트, 시제, 감정
```

R/G/B 값은 **고정 테이블이 아닙니다** — AI가 방향성 확산을 통해 동적으로 매핑합니다:

```
  R ← 대각선 이웃 (↗↘↙↖)    α = 0.05   형태소/의미 관계
  G ← 수직 이웃   (↑↓)       β = 0.08   단어 대체 관계
  B ← 수평 이웃   (←→)       γ = 0.03   구/절 순서 관계
```

### 3. 매칭 파이프라인

2단계 검색: 빠른 조대 필터 → 정밀 스코어링.

```
  입력 절
     │
     ▼
  ┌────────────────────────┐
  │  3-레이어 인코딩         │
  │  + 방향성 RGB 업데이트   │
  └───────────┬────────────┘
              │
       ┌──────▼──────┐       KF < 100: 전체 겹침 스캔
       │  1단계       │       KF ≥ 100: 해시 버킷 → 겹침
       │  조대 필터   │
       │  (겹침 수)   │──── Top-K 후보 (K=8)
       └──────┬──────┘
              │
       ┌──────▼──────┐
       │  2단계       │       RGB 가중 코사인 유사도
       │  정밀 매칭   │       + 블록 스킵 최적화
       │  (코사인)    │──── 최종 매칭 + 유사도 %
       └─────────────┘
```

### 4. 키프레임 / 델타 저장

영상 코덱처럼: I-프레임(전체 스냅샷)과 P-프레임(변화량만 저장).

```
  프레임 0: [I] "귀여운 고양이가 밥을 먹는다."   토픽: 동물_식사
  프레임 1: [P] "귀여운 강아지가 물을 먹는다."   토픽: 동물_식사  (Δ16px)
  프레임 2: [I] "우리는 함께 오랜 세월을 살았다." 토픽: 회상
  프레임 3: [I] "오늘 아침 하늘이 밝다."         토픽: 현재_상태
  ...
  프레임 N: ∞ (프레임 스택으로 무한 컨텍스트)

  ┌──────────────────────────────────────────┐
  │  유사도 ≥ 0.3 → 델타로 저장 (P-프레임)    │
  │  유사도 < 0.3 → 새 키프레임 (I-프레임)     │
  └──────────────────────────────────────────┘
```

### 5. 엔진 최적화

```
  최적화               │ 결과                    │ 가속
 ──────────────────────┼─────────────────────────┼──────────
  1D 정렬 메모리        │ AVX2/SIMD 적용 가능     │  2-5배
  블록 스킵 (16×16)     │ 93% 블록 스킵           │  2-4배
                       │ 정확도 손실 0.000%       │
  적응형 Top-K          │ KF≥100에서 해시 버킷    │ 10-50배
  스파스 델타           │ 16항목 = 96 바이트       │ 메모리
  LRU 캐시 (256 슬롯)   │ 히트율 90%              │  2-10배
 ──────────────────────┼─────────────────────────┼──────────
  종합                  │ 정확도 100% 유지         │ 20-100배
```

## 형태소 분석기

사전 기반 최장일치법. 외부 라이브러리 없이 동작합니다.

```
  입력                  결과
 ──────────────────────────────────────────────────
  "귀여운"          →  [형용사: 귀여운]
  "고양이가"        →  [명사: 고양이] + [조사: 가]
  "밥을"            →  [명사: 밥] + [조사: 을]
  "먹는다."         →  [동사: 먹] + [어미: 는다] + [구두점: .]
```

```
  사전 구성:
  ├── 명사  88개  (동물, 음식, 사물, 자연, 사람, 추상)
  ├── 동사  39개  (먹, 가, 오, 보, 하, 되, ...)
  ├── 형용사 20개  (귀여운, 예쁜, 밝은, 아름다운, ...)
  ├── 조사  26개  (은/는/이/가/을/를/에서/으로, ...)
  └── 어미  20개  (는다/었다/았다/겠다, ...)
```


**필요 환경:** GCC (C11), Make, Linux/macOS/Windows (MinGW)

## 기존 LLM과의 비교

```
                  기존 LLM                  SPATIAL-PATTERN-AI
 ─────────────────────────────────────────────────────────────
  인코딩       토큰 → 벡터 → 행렬        바이트 → 픽셀 → 256×256
  파라미터     고정 크기 행렬              무한 프레임 스택
  컨텍스트     제한된 윈도우 (128K)        무제한 (디스크 한도)
  해석 가능성  불투명한 가중치             히트맵으로 시각 확인
  학습 방식    전체 재학습                 증분 델타 추가
  프레임 메모리  —                        320 KB / 장
```

## 고유 특성

- **해석 가능성**: 히트맵을 열면 AI가 무엇을 기억하는지 눈으로 확인
- **무한 파라미터**: 프레임이 쌓이므로 파라미터 상한 없음
- **무한 컨텍스트**: 디스크가 허용하는 한 무한 확장
- **증분 학습**: 새 데이터는 델타 또는 새 프레임만 추가
- **되감기/분기**: 델타 체인 역추적으로 학습 과정 추적 가능
- **경량성**: 1장 320KB, 엔진 코어 수 MB — Termux, 임베디드에서 동작
